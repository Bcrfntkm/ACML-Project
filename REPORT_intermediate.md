## 1. Архитектура пайплайна

Воспроизведён трёхшаговый пайплайн из статьи ReactionMiner (Zhong et al., EMNLP 2023):

```
PDF
 └─► [SymbolScraper Java] → структурированный текст
       └─► [TopicSegmentor: allenai-specter + C99] → реакционно-релевантные сегменты
             └─► [ReactionExtractor: LLaMA-2-7b + LoRA] → структурированные реакции (JSON)
                   └─► [convert_reactions_to_bio.py] → серебряные BIO-метки
                         └─► [bert/train.py] → дообученный BERT/SciBERT
```

**Ключевое решение:** Мы не обучаем LLaMA заново — используем готовые веса
`MingZhong/reaction-miner-7b-lora` (опубликованные авторами) как учителя для
дистилляции в BERT/SciBERT. Серебряные метки заменяют ручную разметку.

## 2. Что именно извлекаем: сущности и их формат

### 2.1 Схема меток (8 ролей ReactionMiner)

| Метка | Значение | Пример в тексте |
|-------|----------|-----------------|
| **PRODUCT** | Продукт реакции — синтезируемое соединение | `copper(I) piperidide (4)`, `[Sm(hfac)₃Co(acac)₃]` |
| **REACTANT** | Реагент / исходное вещество | `copper(I) mesityl`, `piperidine`, `n-BuLi` |
| **CATALYST** | Катализатор (не расходуется) | `Pd/C`, `Cu(OAc)₂` |
| **SOLVENT** | Растворитель | `tetrahydrofuran`, `DMSO`, `DCM` |
| **TEMP** | Температура реакции | `room temperature`, `125 °C`, `reflux` |
| **TIME** | Время реакции | `5 min`, `6 h`, `overnight` |
| **YIELD** | Выход реакции | `89%`, `74% yield`, `63%` |
| **COND** | Прочие условия | `argon atmosphere`, `nitrogen`, `vacuum` |

### 2.2 Формат BIO (Inside-Outside-Beginning)

Каждый абзац преобразуется в последовательность токенов (whitespace split) с BIO-метками:

```
Preparation  → O
of           → O
copper(I)    → B-PRODUCT     ← начало сущности
piperidide   → I-PRODUCT     ← продолжение
(4):         → I-PRODUCT
A            → O
solution     → O
of           → O
copper(I)    → B-REACTANT
mesityl      → I-REACTANT
...
tetrahydrofuran → B-SOLVENT
...
piperidine   → B-REACTANT
...
room         → B-TEMP
temperature. → I-TEMP
```

Одно предложение может содержать несколько сущностей одного типа
(несколько REACTANT, SOLVENT в двухкомпонентных системах и т.д.).

### 2.3 Конкретные примеры предсказаний SciBERT

**Пример 1:** синтез медного(I) пиперидида

*Текст:* `Preparation of copper(I) piperidide (4): A solution of copper(I) mesityl
(1.462 g, 8.00 mmol) in tetrahydrofuran (10 mL) was treated with piperidine
(3.95 mL, 40.00 mmol) at room temperature.`

| Токен | SciBERT | Золото |
|-------|---------|--------|
| copper(I) | B-PRODUCT | B-PRODUCT |
| piperidide | I-PRODUCT | I-PRODUCT |
| (4): | I-PRODUCT | I-PRODUCT |
| copper(I) | B-REACTANT | B-REACTANT |
| mesityl | I-REACTANT | I-REACTANT |
| tetrahydrofuran | B-SOLVENT | B-SOLVENT |
| piperidine | B-REACTANT | B-REACTANT |
| room | B-TEMP | B-TEMP |
| temperature. | I-TEMP | I-TEMP |

→ **Полное совпадение со всеми 9 размеченными токенами.**

**Пример 2:** C–S сочетание с медным катализатором

*Текст:* `...with 2.0 equiv of copper acetate and DMSO as the solvent,
2-(3-(methylthio)naphthalen-2-yl)pyridine (2b) was obtained as a single product
in 89% yield at 125 °C`

| Токен | SciBERT | Золото |
|-------|---------|--------|
| copper | B-REACTANT | B-CATALYST |
| acetate | I-REACTANT | I-CATALYST |
| DMSO | B-SOLVENT | B-SOLVENT |
| 2-(3-(methylthio)naphthalen-2-yl)pyridine | B-PRODUCT | B-PRODUCT |
| (2b) | I-PRODUCT | I-PRODUCT |
| 125 | B-TEMP | B-TEMP |
| °C | I-TEMP | I-TEMP |

→ SciBERT путает CATALYST и REACTANT для `copper acetate` — это одна из типичных ошибок
(оба класса — «что добавляют в реакцию», граница размыта при малом обучающем корпусе).

**Пример 3:** гетеробиметаллический Sm-Co комплекс

*Текст:* `Starting from [Sm(hfac)₃] (0.29 mmol) and [Co(acac)₃] (0.27 mmol) in toluene
(40 mL), the product [Sm(hfac)₃Co(acac)₃] (74% yield) was recovered.`

| Токен | SciBERT | Золото |
|-------|---------|--------|
| [Sm(hfac)3] | B-REACTANT | B-REACTANT |
| [Co(acac)3] | B-REACTANT | B-REACTANT |
| toluene | B-SOLVENT | B-SOLVENT |
| [Sm(hfac)3 | B-PRODUCT | B-PRODUCT |
| Co(acac)3] | I-PRODUCT | I-PRODUCT |

→ Правильно, но YIELD `74%` не найден (типичная проблема — 74% пишется без слова "yield" рядом).

### 3.3 Золотой тестовый набор (ручная разметка)

#### Как создана золотая разметка

Ручная разметка проводилась в два этапа:

**Этап 1 — выбор абзацев:**  
Из корпуса аннотированных статей отобраны абзацы с высокой плотностью
синтетической лексики (≥4 хитов по шаблону: `mmol`, `mL`, `°C`, `reflux`,
`stirred`, `yield`, названия растворителей и реагентов). Отбирались абзацы из
реальных синтетических процедур, а не обзорных разделов.

**Этап 2 — токен-уровневая разметка:**  
Для каждого абзаца:
1. Текст токенизируется той же функцией `_tokenize()` что и в пайплайне
   (`re.finditer(r'\S+', text)` — whitespace split с сохранением смещений)
2. Читается текст абзаца; для каждого химического выражения определяется роль
   по смыслу (не по тому, что сказала LLaMA)
3. Сущности задаются символьными спанами `(start, end, label)` и конвертируются
   в BIO через ту же функцию `_bio_labels()`, что используется при обучении

**Разграничение REACTANT vs CATALYST:**
- REACTANT: вещество расходуется в ходе реакции (исходные компоненты)
- CATALYST: ускоряет реакцию, не входит в продукт (Pd/C, Cu-соли в каталитических кол-вах)
- Если соль металла используется в стехиометрическом кол-ве — это REACTANT

**Разграничение SOLVENT vs REACTANT:**
- SOLVENT: большой объём (мл), реакционная среда
- REACTANT: малое кол-во (ммоль), расходуется химически

### 5.3 Разбор результатов по классам

**REACTANT (55 сущностей, лучший класс у обоих):**
- SciBERT (F1=0.850) лучше LLaMA (0.753) — студент "жёстче" привязан к токенам текста,
  LLaMA иногда нормализует название реагента (пишет полное IUPAC вместо аббревиатуры),
  что при fuzzy-поиске даёт частичное совпадение, а не полное.

**TEMP (24 сущности):**
- SciBERT (0.889) значительно лучше LLaMA (0.600).
  LLaMA пропускает температуры, выраженные текстом: не всегда тегирует "room temperature",
  "reflux", "0°C" если они вынесены далеко от основного предложения с реакцией.
  SciBERT видит весь контекст абзаца и находит их надёжно.

**TIME (15 сущностей):**
- SciBERT: F1=1.000 — идеальный результат. Паттерн "число + единица" (5 min, 6 h, 24 h)
  выучен почти без ошибок.
- LLaMA: F1=0.750 — пропускает некоторые времена, вынесенные в конец предложения.

**PRODUCT (21 сущность):**
- Оба модели справляются умеренно хорошо; SciBERT имеет более высокий recall (0.857 vs 0.714).
- Главная трудность: сложные химические формулы типа `[Cu(µ-cpna)(phen)(H2O)]n (1)` —
  граница продукта не всегда очевидна.

**YIELD (11 сущностей) — слабое место обоих:**
- LLaMA: F1=0.000. Она часто тегирует численное значение как часть PRODUCT
  ("74% yield") или вовсе игнорирует, если yield написан не как "X% yield", а "afforded
  at X%". LLaMA извлекает yield как структурированное поле JSON, но конвертация спана обратно
  в BIO теряет его (значение "74% yield" не находится точно в тексте).
- SciBERT: F1=0.000. Только 25 B-YIELD токенов в обучающих данных — критически мало.

**CATALYST (2 сущности) — оба 0.000:**
- Золотой набор содержит всего 2 CATALYST-сущности → низкий support → значение неустойчиво.
- Систематическая проблема: граница CATALYST/REACTANT размыта даже для человека.

### 5.4 Ключевые выводы оценки на золоте

1. **SciBERT-студент превосходит учителя (LLaMA) на золотых данных** (+10.4 F1).
   Это контринтуитивный результат, объясняемый тем, что LLaMA работает с более
   высокой точностью (0.765), но низким recall (0.630): она осторожна и пропускает
   ~37% сущностей. SciBERT более полный (recall=0.812).

2. **Серебряные метки смягчают задачу.** Оценка на серебряном test (F1=0.457)
   занижена, т.к. включает шум учителя как "правильные ответы". На честном золоте
   студент показывает F1=0.795.

3. **Ядро системы работает.** На основных классах (REACTANT, SOLVENT, TEMP, TIME)
   SciBERT достигает F1 > 0.85, что достаточно для практического применения.

4. **Узкие места** — YIELD и CATALYST — требуют больше данных и, возможно,
   отдельных эвристик (регулярные выражения для поиска процентов).

## 8. Проблемы и пути улучшения

### П1. Малый объём данных

1 655 серебряных сэмплов — в 3–5× меньше минимума (~5 000) для стабильного BERT-NER.
Линейный тренд: 578 → 1 655 дал +100% F1 на серебре. Целевой объём: 100+ статей → 3 000+ сэмплов.

### П2. YIELD — нулевой F1

25 B-YIELD в обучающих данных (0.5% примеров). Минимум для обучения — 100–200.
Часто yield пишется цифрой без слова "yield" рядом: `(74% yield)` → LLaMA тегирует, но
спан не находится точно.

### П3. CATALYST vs REACTANT — размытая граница

Семантическое различие (расходуется/нет) сложно уловить из текста без
химических знаний. Модель с 216 обучающими примерами CATALYST не может освоить эту границу.

### П4. Шум в серебряных метках (~15–20% галлюцинаций)

LLaMA генерирует значения, не встречающиеся в тексте сегмента. Фильтр в конвертере
отсеивает часть, но не всё. Это подтверждается результатом на золоте: у LLaMA
recall=0.630 — треть реальных сущностей в тексте она пропускает или называет иначе.

### П5. Тестовый split на серебре: F1 = 0.457 занижена

Серебряные метки включают шум учителя. Истинное качество студента (F1=0.795 на золоте)
существенно выше. Для честной оценки следует использовать золотой набор.

## Приложение А: Воспроизводимость

```bash
# Разметка нового PDF
python Science/scripts/run_reactionminer_full.py paper.pdf \
    --output Science/data/annotations/rm_paper.json

# BIO-конвертация (v3 — все 70 статей)
python Science/scripts/convert_reactions_to_bio.py \
    --input Science/data/annotations/rm_*.json \
    --output Science/data/training/rm_ner_v3 --split

# Обучение SciBERT v3
cd Science && python bert/train.py bert/configs/bert_rm_003.yaml --epochs 20

# Обучение лучшей модели (v2, 20 эпох)
cd Science && python bert/train.py bert/configs/bert_rm_002.yaml --epochs 20

# Создание золотой разметки
python Science/scripts/create_gold_full.py

# Оценка на серебряном test
python Science/bert/evaluate.py \
    --model Science/models/bert_rm_002_v2_e20/best_model \
    --test  Science/data/training/rm_ner_v2/test.jsonl

# Оценка на золотом тесте
python Science/bert/evaluate.py \
    --model Science/models/bert_rm_002_v2_e20/best_model \
    --test  Science/data/training/gold_test/gold_full.jsonl

# Benchmark скорости
python Science/scripts/benchmark_inference.py \
    --rm-output Science/data/annotations/rm_molecules-25-01573.json \
    --bert-models Science/models/bert_rm_002_v2_e20/best_model \
    --n-samples 25
```
